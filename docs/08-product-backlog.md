# 제품 기능 백로그

이 문서는 현재 이웃톡의 기능과 아키텍처를 기준으로 다음 기능을 제안한다. 우선순위는 사용자 안전, 핵심 흐름 완성도, 반복 사용 가치, 구현 비용 순으로 판단한다.

## 추천 순서

| 우선순위 | 기능 | 이유 |
|---|---|---|
| P0 | 신고·차단·숨김 | 지역 기반 만남에서 가장 먼저 필요한 안전장치 |
| P0 | 영구 알림함 | 스낵바가 사라져도 매칭·모임·채팅 알림을 다시 확인 |
| P0 | 프로필 온보딩 | `(0, 0)` 위치와 빈 프로필을 방지해 추천 품질 개선 |
| P1 | 모임 일정·대기열 | 실제 만남 전환과 재방문을 높임 |
| P1 | 위치 공개 범위 | 정확한 좌표 대신 동네·거리 구간만 노출 |
| P1 | 매칭 설명·피드백 | 추천 신뢰도를 높이고 알고리즘 개선 데이터 확보 |
| P2 | 관심사 팔로우·저장 | 피드 탐색과 장기 콘텐츠 소비 강화 |
| P2 | 동네 도움·재능 교환 | 이웃톡만의 지역 커뮤니티 정체성 강화 |

## P0: 먼저 구현할 기능

### 신고·차단·콘텐츠 숨김

- 사용자 차단 시 추천, 피드, 모임, 1:1 채팅 생성에서 상호 제외한다.
- 게시물·댓글·메시지·사용자를 신고할 수 있다.
- 신고 사유, 증거 대상, 처리 상태와 관리자 기록을 보관한다.
- `UserBlocked`, `ContentReported` 이벤트를 발행해 검색·추천·알림 모듈이 반영한다.

권장 데이터:

```text
user_blocks(blocker_id, blocked_id, created_at)
reports(id, reporter_id, target_type, target_id, reason, status, created_at, resolved_at)
```

### 영구 알림함

현재 오프라인 알림은 전달 보장용 큐다. 사용자 기능으로는 별도의 읽기 가능한 알림함이 필요하다.

- 최근 알림 목록, 읽음·전체 읽음, 삭제
- 유형별 필터: 매칭, 채팅, 모임, 시스템
- 이벤트 ID 기반 중복 방지
- WebSocket은 즉시 갱신, REST는 복구와 과거 조회에 사용

### 프로필 완성 온보딩

- 가입 직후 관심사, 연령대, 성별 공개 여부, 동네를 단계별로 입력한다.
- 브라우저 좌표를 그대로 저장하기 전에 사용자 확인을 받는다.
- 위치 미입력 사용자는 매칭 시작 대신 온보딩으로 안내한다.
- 완성률과 빠진 항목을 UI에 표시한다.

## P1: 핵심 사용성 강화

### 모임 일정·대기열·리마인더

- 시작·종료 시각, 반복 일정, 장소 안내, 참가 마감
- 정원 초과 시 대기열과 자동 승급
- 하루 전·한 시간 전 리마인더
- 노쇼 기록은 공개 점수가 아니라 운영 보조 신호로만 사용

추천 이벤트:

- `MeetupScheduled`
- `MeetupWaitlisted`
- `MeetupParticipantPromoted`
- `MeetupReminderDue`

### 위치 개인정보 보호

- DB의 정확한 좌표와 API 응답용 공개 위치를 분리한다.
- 화면에는 `1km 이내`, `같은 동네`처럼 구간화된 거리만 제공한다.
- 집 주변 안전 반경과 위치 일시 중지 기능을 제공한다.

### 설명 가능한 매칭

- `공통 관심사 3개`, `선호 거리 이내`처럼 추천 이유를 보여준다.
- 관심 없음 사유를 선택적으로 받아 추천 품질을 개선한다.
- 반복 노출 제한과 일정 기간 다시 보지 않기를 제공한다.

## P2: 차별화 기능

### 동네 도움·재능 교환

짧은 도움 요청이나 재능 나눔을 모임과 분리된 형태로 제공한다.

- 반려동물 산책, 공구 대여, 식물 돌봄, 언어 교환
- 금전 거래보다는 시간·재능 교환 중심
- 완료 확인과 감사 메시지

### 조용한 매칭

즉시 채팅이 부담스러운 사용자를 위해 하루 한 명만 추천하고, 서로 짧은 질문에 답한 뒤 수락하도록 한다.

### 동네 챌린지

- 이번 주 산책 장소 공유
- 동네 쓰레기 줍기
- 독립서점 방문 기록

경쟁 순위보다 협력 달성률과 지역 기여를 중심으로 설계한다.

## 구현하지 않거나 신중해야 할 것

- 정확한 실시간 위치 공개
- 근거 없는 신뢰 점수나 인기 순위
- 동의 없는 연락처 추천
- 자동 생성 메시지를 사람의 실제 발언처럼 표시
- Kafka나 마이크로서비스 자체를 제품 기능보다 먼저 확장하는 일
