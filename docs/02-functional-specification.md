# 기능 명세

상태 표기:

- **구현**: 프론트와 백엔드 흐름이 존재한다.
- **부분 구현**: 핵심 코드는 있으나 계약 불일치나 운영 보완이 필요하다.
- **계획**: 권장 목표이며 현재 동작하지 않는다.

## 공개 화면

| 기능 | 설명 | 상태 |
|---|---|---|
| 랜딩 | 서비스 가치, 가입·로그인 진입 | 구현 |
| 회원 가입 | 이메일, 사용자명, 비밀번호로 가입 후 세션 발급 | 구현 |
| 로그인 | 이메일·비밀번호 확인 후 세션 발급 | 구현 |
| 중복 확인 | 이메일·사용자명 사용 여부 확인 | 구현 |

## 프로필과 인증

| 기능 | 규칙 | 상태 |
|---|---|---|
| 세션 인증 | `X-Session-Id` 헤더로 사용자 식별 | 구현 |
| 로그아웃 | Redis 세션 삭제 | 구현 |
| 내 정보 | 현재 세션의 사용자 반환 | 구현 |
| 프로필 수정 | 사용자명, 나이, 성별, 소개, 위치, 관심사 수정 | 구현 |
| 프로필 사진 파일 업로드 | JPG·PNG·GIF·WebP 파일을 받아 최대 1024px WebP로 최적화하고 교체 시 이전 파일 삭제 | 구현 |
| 보호 라우트 | 미인증 사용자를 `/login`으로 이동 | 구현 |

프로필 완료 조건은 나이, 성별, 관심사, 위도·경도, 주소가 모두 채워지는 것이다.

## 피드

| 기능 | 규칙 | 상태 |
|---|---|---|
| 피드 조회 | 페이지 0부터 조회, 크기 1~50 제한 | 구현 |
| 게시물 작성 | 사진·동영상 최대 10개, 순서, 설명, 관심사 태그 저장 | 구현 |
| 미디어 검증 | 파일 시그니처 검사, 사진 10MB·동영상 100MB·전체 200MB 제한 | 구현 |
| 미디어 최적화 | 정적 이미지는 WebP, 동영상은 MP4(H.264/AAC)로 변환하고 WebP 썸네일·크기·재생시간 저장 | 구현 |
| 미디어 표시 | 이미지 캐러셀과 브라우저 구간 재생을 지원하는 동영상 플레이어 | 구현 |
| 미디어 정리 | 게시물 트랜잭션 커밋 후 로컬 원본 파일 삭제 | 구현 |
| 좋아요 | 사용자와 게시물 조합당 한 번 | 구현 |
| 좋아요 취소 | 본인의 좋아요 제거 | 구현 |
| 댓글 조회·작성 | 게시물별 댓글 목록 및 작성 | 구현 |

## 관심사 매칭

| 기능 | 규칙 | 상태 |
|---|---|---|
| 선호 조건 | 최대 거리, 나이 범위, 성별, 관심사 저장 | 구현 |
| 추천 | 현재 사용자와 조건이 맞는 프로필 반환 | 구현 |
| 직접 요청 | 특정 사용자에게 매칭 요청 | 구현 |
| 수락 | 매칭 상태 변경 후 1:1 채팅방 반환 | 구현 |
| 거절 | 매칭 상태를 거절로 변경 | 구현 |
| 만료 | 대기 매칭은 생성 후 24시간 기준 | 구현 |
| 주변 검색 | 위도, 경도, 반경으로 사용자 검색 | 구현 |

매칭 정확도는 거리와 프로필·관심사 데이터를 기반으로 계산한다. 위치값이 기본값 `(0, 0)`인 신규 계정은 추천 품질이 낮을 수 있다.

## 취미 모임

모임은 별도 `Meetup` 엔티티가 아니라 공개 `GROUP` 채팅방을 재사용한다.

| 기능 | 규칙 | 상태 |
|---|---|---|
| 모임 목록 | 키워드·관심사·페이지 조건으로 공개 방 검색 | 구현 |
| 모임 생성 | 설명, 태그, 위치, 최대 인원을 가진 공개 그룹방 생성 | 구현 |
| 참가 | 최대 인원과 중복 참가 규칙 적용 | 구현 |
| 탈퇴 | 참가자에서 제거 | 구현 |

## 채팅

| 기능 | 규칙 | 상태 |
|---|---|---|
| 채팅방 생성·조회 | 1:1 또는 그룹방, 페이지 검색 | 구현 |
| 입장·퇴장 | 참가자 집합 변경, 생성자 퇴장은 방 삭제 | 구현 |
| 메시지 조회 | 방 참가 권한을 확인한 뒤 페이지 반환 | 구현 |
| 메시지 전송 | REST 또는 STOMP로 저장·전달 | 구현 |
| 채팅 사진·동영상·파일 첨부 | 메시지당 최대 5개, 미리보기·진행률·실시간 메타데이터·다운로드 지원 | 구현 |
| 채팅 첨부 검증 | 이미지 10MB, 영상 100MB, 문서 25MB, 전체 120MB 및 파일 시그니처 검사 | 구현 |
| 채팅 첨부 정리 | 방 삭제 트랜잭션 커밋 후 원본과 썸네일을 함께 삭제 | 구현 |
| 읽음 처리 | 메시지별 또는 방 전체 읽음 사용자 기록 | 구현 |
| 실시간 갱신 | 사용자별 큐로 메시지와 방 목록 갱신 | 구현 |
| 방 수정·종료 | 생성자만 이름·설명·정원·상태 수정 | 구현 |
| 내 방·그룹 검색 | 프론트 호환 목록 API | 구현 |
| 1:1 방 열기 | 기존 방 재사용 또는 새 방 생성 | 구현 |

## 도메인 이벤트와 Outbox

| 기능 | 규칙 | 상태 |
|---|---|---|
| `MatchCompletedEvent` | 양측 수락과 채팅방 생성 후 같은 트랜잭션에 기록 | 구현 |
| `MeetupJoinedEvent` | 신규 참가자 저장과 같은 트랜잭션에 기록 | 구현 |
| 커밋 후 전달 | `ApplicationEventPublisher`로 타입 이벤트 전달 | 구현 |
| 장애 재시도 | 5초마다 미발행 이벤트 최대 100개 재처리 | 구현 |
| 보존 정리 | 발행 후 7일이 지난 이벤트를 매일 정리 | 구현 |

## 비기능 요구사항

- 비밀번호는 평문으로 저장하지 않는다.
- 인증이 필요한 API는 세션 검증을 거친다.
- 프론트와 백엔드는 UTF-8을 사용한다.
- 컨테이너는 헬스체크를 제공한다.
- PR은 타입 검사, 테스트, 빌드, 컨테이너 빌드를 통과해야 한다.
- 운영 비밀은 환경 변수로 주입한다.
